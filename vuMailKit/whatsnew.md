# What's New in vuMailKit

Build 1.51 update

Release date: 2026-04-03

This page highlights the most recent improvements in the current vuMailKit Basic edition.

## Recent highlights

- Added improved Gmail OAuth success page handling for a cleaner post-authorization experience
- Added the new **OAuth Success Name** setting in the global template for Google OAuth
- Expanded template guidance so developers know where to enter the name shown on the Google success page
- Continued clarification of Google OAuth setup requirements and related documentation links
- Clarified supported Clarion version coverage and installer support for older Clarion environments

## Major function and setup areas affected

| Area | Notes |
|------|-------|
| Google OAuth | Gmail authorization now supports a better-looking success page after successful OAuth completion. |
| Template Settings | The global template now includes **OAuth Success Name**, which lets the developer control the name shown on the Google success page. |
| Documentation | Template usage and related OAuth setup pages now explain where this setting is entered and what it affects. |
| Compatibility | Documentation now reflects active installer support for Clarion versions back to Clarion 5.5. |

## Earlier documentation refresh highlights

- Added licensing initialization and license-diagnostics documentation
- Added DEMO-mode documentation and restrictions
- Clarified managed profile save, load, and immediate-apply behavior
- Expanded the function-level documentation for autodetect, deep scan, POP3, globals, logging, and profile workflows
